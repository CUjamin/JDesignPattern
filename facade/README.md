外观模式
    
    为了系统中的一组接口提供一个已知的界面，此模式定义了一个高层接口
    这个接口使得这一子系统更加容易使用
    
    
外观模式的使用

    首先，在设计初期阶段，应该要有意识地将不同的两个层分离。
    例如数据访问层和业务逻辑层之间、业务逻辑层和和表示层之间，建立外观Facade
    
    其次，在开发阶段，子系统往往因为不断的重构演化而变得越来越复杂，
    大多数模式使用时也都会产生很多很小的类，这本身是好事，但却给外呼调用他们的用户程序带来使用上的困难。
    增加外观Facade，可以提供一个简单的接口减少他们之间的依赖。
    
    第三，维护一个遗留的大型系统时，可能这个系统已经非常难以维护和扩展了，但因为它包含了很多非常重要但功能，新的需求开发必须要依赖于它。
    此时使用外观Facade也非常合适。为新系统开发一套外观Facade类，来提供设计粗糙或高度复杂但遗留代码但比较清晰简单的接口，
    让新系统与Facade对象交互，Facade与遗留代码交互所有复杂的工作。
